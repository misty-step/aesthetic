# aesthetic — vision

The durable north star. The law and the visual identity live in
`aesthetic.css`, `tokens.json`, and `DESIGN.md`; the agent contract lives in
`CLAUDE.md`. This file is the product strategy those serve — the frame that
ranks the backlog (`backlog.d/011`–`016`) and decides what is in and out.

## One sentence

One CSS file that makes any product — built by a human or generated by an
agent — come out unmistakably in-law, and that can _prove_ it did.

## Audience and job

Two consumers, one product:

- **The pinning developer.** Pins a git tag, gets a stylesheet, steers the
  accent, ships a product that reads like the family. Job: a design system
  that is correct by default and never needs a designer in the loop to stay
  correct.
- **The adopting agent.** Reads the catalog, the tokens, and the registry;
  composes a view; needs the law to be _legible and machine-checkable_, not
  buried in prose it will weight below a nearby example. Job: generate
  in-law UI cold, and fail loudly when it drifts.

Both are served by the same artifact. The agent is not a second-class
consumer — it is the one the 2026 category is being rebuilt for.

## Category

A **law-governed, LLM-safe design system**. The cresting 2026 wave (Polar
Orbit, Builder.io, Atlassian DESIGN.md, Pandya, the rhythmguard/token-guard
stylelint plugins) converges on one insight: when a model authors the UI,
prose rules rot and the interface drifts toward "a thousand slightly
different grays" exactly where the system is silent. The answer is not more
documentation — it is a **constrained vocabulary plus a deterministic gate**.
aesthetic already embodies this; the backlog finishes it.

## Strategic bets

- **Bet B, not Bet A.** Constrain the vocabulary and enforce it with a gate
  (Bet B). Do _not_ adopt typed-prop / compiler / ban-the-`<div>` / StyleX
  authoring-time enforcement (Bet A) — it detonates the zero-build,
  single-file, framework-free identity, and the market has not moved there
  (Tailwind ~12M weekly vs StyleX ~100–300K). A typed React companion is a
  possible-future package only, never the product.
- **Gate the render, not the source.** Everyone else lints source
  (framework-locked, authoring-time). aesthetic can ship a _render_ gate
  (framework-agnostic, output-time) — the same property that lets it be one
  zero-build file. This is the differentiated edge. (Epic 015.)
- **Explorer, not Storybook-the-tool.** "A proper component library" =
  finish the catalog into a world-class, _zero-build_ explorer (per-component
  pages, a rendered token reference, state matrices, copy-code, a composition
  feed). Storybook the tool — a bundler, an MDX pipeline, a JS controls
  panel, framework stories — is rejected; its _jobs_ are delivered natively.
  (Epic 016.)
- **The system is its own proof.** `site/` rides HEAD and is built only with
  the system. If the canonical dashboard, workbench, lab, or token reference
  cannot be built from primitives with no escape hatch, no consumer can —
  that is the acceptance bar for every channel the kit names.

## Non-goals

- No build step, no framework, no runtime in the product.
- No component library as the product (classes + recipes are the library).
- No token tiering / aliasing layer until a consumer needs alias-recoloring
  — the flat, intent-named token set is deliberate, not unfinished.
- No new font size, no serif display face, no ambient motion, ever.

## What excellent looks like (6–12 months)

1. **The fleet resembles.** The seven misty-step UI consumers (011) all ride
   aesthetic, steered to their own personality, recognizably one family.
2. **The law travels.** A consumer's CI runs the shipped render-gate; an
   off-law page fails and names the offender (015). The law is enforced on
   surfaces this repo never sees.
3. **The system is explorable.** The catalog is a world-class zero-build
   explorer — every primitive in every state, the tokens rendered, the markup
   one click from a consumer's clipboard, a machine feed an agent composes
   from without scraping HTML (016).
4. **Density and the lab are in-kit.** The workbench, comparison matrix, and
   measurement instruments (012–014) are all composable from primitives — the
   kit serves a dense agent lab, not just a marketing page.
