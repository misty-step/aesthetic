Adopting aesthetic

One request, one file, no build step. aesthetic.css carries the whole system — tokens, registers, screens, forms, instruments — and tokens.json rides along for tooling that wants the values raw. Consumers pin a git tag and upgrade deliberately; the file never moves under you. 

pnpm add github:misty-step/aesthetic#v2.8.0

Adoption is four small moves:

  1. Link the stylesheet, ahead of your own styles.
  2. Load Geist and Geist Mono.
  3. Boot the mode in <head>, before first paint.
  4. Steer the scheme — the accent is yours.

Link the stylesheet

Reference aesthetic.css with a plain <link rel="stylesheet"> before any page glue, so your local layout rules can lean on the tokens. There is nothing to compile and nothing to configure; the import is the install. Vendoring the file works exactly the same — it has no dependencies.

Load Geist

The system sets --ae-font to Geist and --ae-font-mono to Geist Mono, falling back to Helvetica and the platform monospace until they arrive. One Google Fonts request covers both families:

  • Geist:wght@100..900 — the text face; the system spends exactly three weights: 400, 550, 800.
  • Geist+Mono:wght@100..900 — tables, tags, figures, code.
  • Self-hosting works the same; only the family names matter.

Boot the mode

Light and dark are equals, defaulting to the system preference with a user toggle. Inline a tiny script in <head> that reads the persisted ae-mode choice and sets the root class before first paint, so the page never flashes the wrong mode. The optional recipes.js carries the toggle itself and the rest of the behavior glue.

The change is one soft 700ms breath — a view transition where supported, a uniform color ease where not, instant under reduced motion.

Steer the accent

Identity comes from ink, hairlines, type, and motion; the accent is yours. Override the scheme dials at :root after the import and every surface follows — see the volume knob for six schemes riding one fragment.

Each dial is a pair — --ae-accent / --ae-accent-dark, and likewise for the status triplet. Everything not listed — ink, wash, line, type — is the shared identity; do not steer it.

The law itself lives in the header of the file and in tokens.json; the canonical markup for every primitive is in the catalog — view-source is documentation.
